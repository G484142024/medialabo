「Webコンテンツ作成」の成果

本テーマについて

このテーマでは、HTMLでページの形を作り、CSSでデザインを整え、JavaScriptで動きをつける基本を学びました。また、調べたいことをリファレンスで確認しながら、 画像や音声を使って自分なりのWebページを作る力も身につけることが本テーマとなりました。

プロジェクト課題について

今回のプロジェクトでは、インターネット上のサーバにアクセスして情報を検索し、その結果をWebページに表示するという課題が出されました。テーマは「NHKの番組表」「グルメ情報」「世界の天気」の3つから選ぶ形式で、 私は「世界の天気」を選びました。ユーザーが都市のIDを入力して検索ボタンを押すと、その地域の現在の天気（天気の説明や気温など）が画面に表示されるしくみを作りました。検索が終わったあとは、 最初の入力画面から自動的に結果を表示する画面に切り替わるように工夫しました。．

プロジェクト課題

数当てゲーム

課題3-1では、JavaScriptの基本的な文法を使って、1から10までのランダムな数字を当てる数当てゲームを作成しました。 ユーザーは最大3回まで予想を入力でき、予想した数字が正解かどうかだけでなく、「もっと大きい」「もっと小さい」といったヒントもコンソールに表示されるようにしています。 また、もしユーザーが最初の予想で正解した場合は、次の予想入力時にゲームが終了したことを表示するようにしています。 しかし、今の時点ではまだ完成していないため予想数が４として固定して論理演算だけ終わらせました。ページ上にもまだ途中な作業だけです。 課題4-1では、実際にDOMやイベント処理などを使うことによって、数当てゲームを完成しました。予想数を入れてボタンをしてたら上手く動きました。また、Webページ上にもちゃんとできました。

各授業愛の学習事項

第1回 WWW と HTML

今回の学習で、HTMLのいくつかの基本的な要素を使ってWebページの構造を作る方法を学びました。ハイパーリンクや画像の表示、説明リストなどを使うことで、 より分かりやすいページが作れると感じました。また、headやbodyといった要素を使うことで、ページ全体の構成がきちんと整理できるようになりました。

演習の成果:　演習1-4:HTMLのさまざまな要素

第2回 CSS による WEBページの装飾

今回の学習では、CSSを使ってHTMLで作ったWebページの見た目を整え、より見やすくきれいにする方法を学びました。CSSにはたくさんのプロパティがあるということが分かりました。 背景やリンクの色を変えたり、border・font・paddingなどのスタイルを設定することで、ページがよりWebページらしくになりました。

演習の成果:　演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

今回の学習で、JavaScriptを使えばWebページにいろいろな動きや機能を追加できることが分かりました。Javaとは違って、JavaScriptはコンパイルなしで1行ずつ実行されるのが特徴です。 また、HTMLとは別にJavaScriptのファイルを作って読み込む方法もあり、その際はsrc属性やdefer属性を使う必要があることも学びました。

演習の成果:　演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

今回、DOMの仕組みを学び、HTMLの要素を探したり変更したりできることがわかりました。フォームの操作やJavaScriptのイベント処理も理解でき、 これらを組み合わせてWebページの動きをよりスムーズにする方法を身につけました。

演習の成果:　演習4-10: 足し算ページ

第5回 イベント処理

今回もDOMの機能をさらに学び、要素の追加や編集、削除ができるようになりました。ボタンを押すとWebページの内容が変わる操作を作り、よりリアルなページができました。 今は検索結果が一度だけ表示されますが、これから実際の情報を取得してページを完成させたいことができると思います。

演習の成果:　演習5-7: DOMプログラミング(2)

工夫点，苦心点，強調点，感想

今回のプロジェクトで私が工夫した点としては、Webページがしっかり動くようにすることでした。ただ見た目を整えるだけでなく、ユーザーが検索ボタンを押したら画面がスムーズに切り替わり、 結果がきちんと表示されるように注意しました。HTML , CSS と JavaScriptを組み合う焦ることで、実際に動いているWebページを作成しました。また、検索結果がわかりやすくなるように文字の大きさや色も工夫しました。 これによって、使う人が迷わず操作できるページを目指しました。苦労した点は、やはりJavaScriptの操作でした。HTMLやCSSは主に見た目を整える役割ですが、JavaScriptはWebページに動きをつけたり、ユーザーの操作に応じて処理を行ったりするため、 考えることが多くとても大変でした。私はJavaScriptを使うのが初めてだったため、コードの書き方や動きの仕組みがよくわからず、最初はかなり混乱しました。特に、関数の使い方やイベント処理の考え方を理解するのに時間がかかりましたし、 どのように処理が順番に実行されるのかをつかむのも難しかったです。それでも、少しずつ慣れていくことができました。今回のプロジェクトで特に強調したいのは、「シンプルさ」を意識してWebページを作ったことです。初めてJavaScriptを使ったので、複雑なことをいきなりやるのではなく、 できるだけ見た目も操作もわかりやすいページにしようと考えました。検索ボタンや入力欄の配置、結果表示の場所もシンプルになるよう工夫しました。表示される天気の情報もテーブルの形にしたのでより見やすいのではないかと思います。また、色使いも落ち着いたものにして、 必要な情報だけを表示するようにしました。結果として、使いやすくてすっきりとしたWebページが完成し、自分でも満足になったと思います。今回のプロジェクトを通して、HTML、CSSとJavaScriptがそれぞれどんな役割を持ち、どう組み合わせてWebページが動いているのかをしっかり理解することができました。 初めて、自分のWebページを作成したこの経験がとても良かったと思います。様々な新しいことや情報が得られます。中でも、JavaScriptでページの内容を動的に変えられるところがとても面白く感じて、もっと勉強してみたいと思いました。これからは、この経験を活かして、もっと複雑な機能のあるWebアプリにもチャレンジしていきたいです。
